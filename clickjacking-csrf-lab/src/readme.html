Clickjacking + CSRF token

Goal: delete your account by luring a victim to a decoy page that frames /account.php and routes their click onto the hidden Delete button. CSRF token will not help because the click happens inside the framed page under the victim's session.

  1. Run the lab and login at /login.php with alice / alice123.
  2. Host the attacker HTML from a different origin (e.g., python3 -m http.server 8000).
  3. Craft HTML that overlays a near-transparent iframe of /account.php over a big decoy button that says CLICK.
  4. When the victim clicks, the hidden Delete button receives the click and the account is deleted.
